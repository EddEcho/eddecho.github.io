About

Making cloud problems reproducible

I'm a support engineer who thinks like a developer. My day-to-day is taking a vague bug report — "Terraform fails against LocalStack" — and turning it into a minimal, pinned, reproducible test case that engineering can fix in one pass.

Before LocalStack I worked across Liferay, Jenkins, and enterprise Java environments, which taught me how real production systems break: slowly, and in the configuration layer.
